🔹 1. Variables

👉 What

A variable is like a storage box where you can save a name, number, or any data to use it later.

👉 Why

So you don't have to type the same data again and again. You can just use the box name instead.

👉 When

Use const for things that never change (like your birthday) and let for things that update. 

let name = "Nishant"; // Declaring a variable that can be changed later
const age = 21;       // Declaring a constant piece of data that stays fixed
var city = "Delhi";   // Old way of declaring variables (generally avoided today)

🔹 2. Hoisting

👉 What

Hoisting is when JavaScript moves your variable and function names to the top before running the code.

👉 Why

It helps the code run smoothly even if you use a function before you actually write it down.

👉 When

This happens automatically in the background every time your code starts running. 

console.log(a); // Logs 'undefined' because 'var a' declaration is hoisted to the top
var a = 10;      // Now the value 10 is actually assigned to the variable

// Un-commenting below would throw a ReferenceError due to Temporal Dead Zone
// console.log(b); 
// let b = 20;

🔹 3. Variable Scope

👉 What

Scope is a rule that decides which parts of your code can "see" and use a specific variable.

👉 Why

It keeps your code clean and safe so different parts don't accidentally mess up each other's data.

👉 Types

Global (seen by all), Function (only for one function), and Block (only inside {}). 

let globalVar = "global"; // Lives in the highest level (Global Scope)

function test() {
  let funcVar = "private"; // Only exists inside this function (Function Scope)
}

if(true) {
  let blockVar = "inside"; // Only exists inside these curly braces (Block Scope)
}

🔹 4. Functions & Arrow Functions

👉 What

A function is a "package" of code that does a specific job when you call its name.

👉 Why

You can write the logic once and use it many times, making your code much shorter and cleaner.

👉 Tip

Use "Arrow functions" in modern code because they are shorter and easier to type. 

// Standard function for adding two numbers
function add(a, b) {
  return a + b; // Result is returned to the caller
}

// Same logic written as a modern, shorter Arrow Function
const addArrow = (a, b) => a + b; // Arrow functions have implicit return

🧠 Closures PRO INFO

👉 What

A closure is like a function with a "memo" that remembers variables from its birthplace, even after the function moves away.

👉 Why

It helps keep your data safe and private inside a function so nobody else can change it by mistake.

👉 When

Use it when you want to create a "private" variable that only certain functions can touch. 

function createCounter() {
  let count = 0; // Private variable initialized in outer scope
  return {
    increment: () => ++count, // Accesses and modifies 'count' via closure
    getCount: () => count     // Accesses 'count' without modifying it
  };
}

const counter = createCounter();       // Outer function runs and returns object
console.log(counter.increment()); // 1  // Inner function still has access to 'count'
console.log(counter.increment()); // 2  // State is preserved across calls
console.log(counter.getCount());  // 2  // Final value of 'count' is retrieved

🔹 5. Data Types

👉 What

Data types are just "categories" of values like words (string) or numbers (number).

👉 Why

It tells the computer how to use the value (like adding numbers or joining two words).

👉 Types

Simple ones (like 10 or "A") and complex ones (like a list of names or an object). 

let str = "Hello";      // String data type (text)
let num = 10;           // Number data type (digits)
let bool = true;        // Boolean data type (true or false)
let arr = [1, 2];       // Array (ordered list of values)
let obj = { name: "N" }; // Object (collection of key-value pairs)

🔹 6. Operators

👉 What

Operators are special symbols like +, -, or = that let you do math or compare values.

👉 Why

They are the tools you use to build logic, like "if points are more than 10, then win".

👉 Tip

Use === to be very safe when checking if two things are exactly the same. 

let a = 5, b = 2;       // Initialize two number values
console.log(a + b);      // Addition: Adds 5 and 2 (Result: 7)
console.log(a > b);      // Comparison: Checks if 5 is greater than 2 (true)
console.log(a === "5");  // Strict Equity: Checks value (5) AND type (Number vs String) (false)

🔹 7. Conditional Statement

👉 What

Conditionals are like a "fork in the road" that tells the code to go left or right based on an answer.

👉 Why

Your app needs to make decisions, like "if the password is correct, then log the user in".

👉 Tip

Use if/else for regular checks, or the ? : (ternary) for very quick one-line answers. 

let age = 18;          // Sample value for testing
if (age >= 18) {       // Logical test: If age is 18 or above
  console.log("Adult"); // This runs if the condition is TRUE
} else {
  console.log("Minor"); // This runs if the condition is FALSE
}

// Ternary shorthand for simple assignments
let status = age >= 18 ? "Adult" : "Minor"; // [condition] ? [if true] : [if false]

🔹 8. Switch

👉 What

A Switch is a cleaner way to handle a long list of "if" checks when you have many different options.

👉 Why

It's much easier to read than writing 10 different if/else statements in a row.

👉 Tip

Don't forget the break keyword, otherwise your code will keep running into the next option! 

let day = 1;      // Input value to match against cases
switch (day) {
  case 1: 
    console.log("Mon"); // Runs if day strictly equals 1
    break;              // Stops the switch from running into the next case
  default: 
    console.log("Other"); // Runs if no cases match the input
}

🔹 9. Looping

👉 What

A loop is used to repeat the same work over and over again automatically until it's finished.

👉 Why

Instead of typing the same code 100 times, you can just write one loop to do it all for you.

👉 Types

for (count 1 to 10), forEach (for every item in a list), for...of (simpler list loop). 

// Count-based loop: starts at 0, runs while i < 3, increments i each time
for (let i = 0; i < 3; i++) console.log(i); 

let j = 0;
while (j < 3) { j++; } // Condition-based loop: runs as long as j < 3

// Modern 'for...of' loop: extracts the actual values directly from an array
for (let value of [10, 20]) console.log(value);

🔥 10. Array Methods

👉 What

Array methods are built-in "superpowers" for lists that let you change or find data very quickly.

👉 Why

They handle all the hard work of searching and updating lists so you don't have to write complex code.

👉 Tips

map changes everything in a list; filter picks only specific items you want. 

const fruits = ["Apple", "Banana"]; // Starting with a simple list

fruits.push("Mango");   // Adds "Mango" to the END
fruits.pop();           // Removes the LAST item
fruits.shift();         // Removes the FIRST item
fruits.unshift("Kiwi"); // Adds "Kiwi" to the START

// slice: Takes a piece of the list (from index 0 to 2)
const someFruits = fruits.slice(0, 2); 

console.log(fruits.length); // Tells you how many items are in the list

// map: Changes every item (e.g., lowercase to UPPERCASE)
const loudFruits = fruits.map(f => f.toUpperCase()); 

// filter: Keeps only items that match a rule
const longNames = fruits.filter(f => f.length > 5);

🔹 11. Object

👉 What

An object is a way to group related data together, like a "User Profile" hosting a name and age.

👉 Why

It's much better than having 10 separate variables for one user. Everything stays in one "box".

👉 Parts

Properties are the facts (like age); Methods are the actions (like greet()). 

let user = {
  name: "Nishant",  // A property (fact)
  age: 21,
  greet() { 
    console.log("Hello!"); // A method (action)
  }
};

// Main Object Methods:
Object.keys(user);   // Returns all names: ["name", "age", "greet"]
Object.values(user); // Returns all data: ["Nishant", 21, function]
Object.entries(user);// Returns pairs: [["name", "Nishant"], ["age", 21]]

🔹 12. Destructuring

👉 What

Destructuring is a shortcut to quickly "pull out" values from arrays or objects into their own variables.

👉 Why

It saves you from writing let name = user.name over and over again. It's much faster!

👉 Usage

Used a lot in modern libraries like React to handle data in a very clean way. 

// 1. Array Destructuring (order matters)
const colors = ["Red", "Green", "Blue"];
const [first, second] = colors; // first="Red", second="Green"

// 2. Object Destructuring (name matters)
const person = { name: "Nishant", age: 21 };
const { name, age } = person; // Extracts 'name' and 'age' into variables

🔹 13. Local Storage

👉 What

Local Storage is like a "small locker" in your browser where you can save data even after you close the tab.

👉 Why

It's perfect for saving user settings like "Dark Mode" so the website remembers them next time they visit.

👉 Limit

You can only store a small amount of text data (around 5MB), so don't put big files here! 

localStorage.setItem("user", "Nishant"); // Saves 'Nishant' under the key 'user'
let name = localStorage.getItem("user");  // Retrieves the value for 'user'
localStorage.removeItem("user");         // Deletes only the 'user' entry
localStorage.clear();                    // Wipes all data from current domain storage

🔹 14. Exception Handling

👉 What

Exception handling is a "safety net" that catches errors so your whole website doesn't crash.

👉 Why

Mistakes happen (like a bad API call). This lets you show a friendly message instead of a blank screen.

👉 Pattern

Use try for the risky code, catch to handle the error, and finally for the cleanup. 

try {
  let x = y; // This will fail because 'y' hasn't been defined
} catch (err) {
  console.log("Error: " + err.message); // Execution jumps here on failure
} finally {
  console.log("Cleanup"); // This block runs regardless of success or failure
}

🔹 15. Sync vs Async

👉 What

Sync code runs "one-by-one" in order. Async code lets the computer do other work while waiting for a task.

👉 Why

It stops your website from "freezing" while waiting for a slow task like downloading an image.

👉 Real Life

Sync is like a queue at a bank; Async is like a restaurant pager that rings when your food is ready. 

console.log("Start"); // Runs immediately (Synchronous)

// Runs in the background, executes the arrow function after 1000ms
setTimeout(() => console.log("Async"), 1000); 

console.log("End");   // Runs immediately after "Start", before "Async"

🔹 16. Promises & Async/Await

👉 What

A Promise is like an "IOU" note. It promises that you will get a result (Success or Failure) later on.

👉 Why

It makes writing "waiting" code much easier than using old-fashioned callbacks.

👉 Modern View

Use async/await because it makes your code look clean and simple, like human sentences. 

// Async function waits for a Promise to resolve without freezing the page
async function getData() {
  const res = await new Promise(r => r("Done")); // Pauses here until 'r' is called
  console.log(res); // Prints "Done" after the promise is fulfilled
}
getData(); // Initiates the async operation

🔹 17. DOM Manipulation

👉 What

The DOM is like a "Remote Control" for your website that lets JavaScript change HTML and CSS.

👉 Why

Without the DOM, your website would be static. This is how you make buttons work or show pop-ups.

👉 Example

Clicking a "Like" button and seeing the heart turn red is possible because of the DOM. 

const btn = document.getElementById("btn"); // Finds the HTML element with ID 'btn'

// Listens for a 'click' event and runs the function when it occurs
btn.addEventListener("click", () => {
  btn.style.backgroundColor = "blue"; // Directly changes the element's CSS
  alert("Interactive!");            // Triggers a browser pop-up
});

🔹 18. ES6 vs ES7+

👉 What

ES6 and ES7 are just "New Versions" or updates to the JavaScript language.

👉 Why

They bring new shortcuts and features that make your life as a developer much easier and faster.

👉 Summary

ES6 brought big changes (Arrow fns, Classes); ES7+ added smaller helpful tools (Includes). 

// ES7 Features
const hasTwo = [1, 2].includes(2); // Simple check if element exists (true)
const cube = 2 ** 3;              // Exponentiation shorthand: 2 to the power of 3 (8)

// ES6 Template Literal
console.log(`The result is ${cube}`); // Injects code directly into a string

📦 ES Modules EXTRA

Organize your code into reusable files using import and export statements. 

// math.js
export const add = (a, b) => a + b; // Exposes this function to other files
export default function multiply(a, b) { return a * b; } // Main export

// app.js
import multiply, { add } from './math.js'; // Pulls logic from external files
console.log(add(10, 5)); // Uses imported function

💡 E-Commerce Simulation

A practical demonstration of combining several JS concepts to manage a virtual shopping cart. 

const cart = [
  { name: "Laptop", price: 1000, qty: 1 }, // Defining an array of objects
  { name: "Mouse", price: 50, qty: 2 }     // Each object represents a product
];

// sum starts at 0, adds (price * qty) for every item in the cart
const total = cart.reduce((sum, item) => sum + (item.price * item.qty), 0);

// Creates a new array with only items costing more than $100
const premium = cart.filter(item => item.price > 100);

// Loops through the premium list and logs just the names
premium.forEach(({ name }) => console.log(`Premium item: ${name}`));
